## 1. Why This Hackathon

The page argues that the age of autonomous AI agents is already unfolding and positions the event as a builder-focused response.

Stated goals:

- demystify AI through direct experience with capable open-source models
- experience open source as infrastructure for equitable innovation
- explore upstream by engaging not just with applications, but with models, platforms, and interaction paradigms underneath them

## 3. Seven Themes

### Theme 1: Agents Meet Apps

Applications are described as moving from static tools to living systems that anticipate needs, coordinate with other agents, and evolve with users. The page explicitly says the boundary between app and agent will blur.

Possible directions:

- Agentic X with autonomous posting and curation
- Agentic WeChat with intelligent message management
- Agentic Spotify with taste-aware playlist agents
- any application reimagined in agent-first form

### Theme 2: Claws and Octos

This section frames OpenClaw as an open-source paradigm for personal AI assistants and Octos as an OS layer for family/team agent experiences. The emphasis is on upstream contribution: skills and plugins created here can continue living in the broader open-source ecosystem.

Possible directions:

- practical skills for families and teams
- cross-platform agent capabilities
- security and permission management skills
- creative skills such as games, music generation, data visualization, and generative art

### Theme 3: Human-Agent Interaction

The page places this theme in the lineage from command line to GUI to touch to voice, then argues that proactive agents may redefine the concept of interface itself.

Possible directions:

- beyond-chat interaction paradigms
- adaptive interfaces
- multi-agent collaborative interfaces
- interface-less or ambient agents

### Theme 4: Education

The page ties this theme to Socrates, Feynman, and Piaget, emphasizing learning through questioning, explanation, and adaptation rather than one-way answer delivery.

Possible directions:

- Socratic AI tutor
- Feynman learning partner
- adaptive learning systems
- immersive history or science exploration agents

### Theme 5: Content Remixing

The central claim is that true creativity is inherently cross-modal, so future AI content systems should orchestrate multiple modalities rather than keep them siloed.

Possible directions:

- cross-modal content creation pipelines
- AI-driven interactive narrative
- intelligent presentation generation
- scientific visualization remix workflows

### Theme 6: Productivity

The page positions agents as moving from assistance to autonomy in repetitive, rule-driven, system-spanning knowledge work.

Possible directions:

- business process automation
- knowledge management agents
- meeting agents
- multi-system orchestration agents

### Theme 7: Agents with Voices

Voice is framed as the most natural interaction layer and as a path for agents to move from screens into the physical world.

Referenced stack:

- EchoKit
- Rust-based
- open-source
- native MCP support
- model-agnostic connection layer

Possible directions:

- voice-driven personal assistants
- multilingual real-time translation agents
- voice-controlled tool orchestration
- AI characters with distinct personalities
- voice agents linked to IoT devices
